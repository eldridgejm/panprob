Renderers

panprob converts problems by first parsing them into an intermediate format (an abstract syntax tree, or AST, of nodes from panprob.ast) and then rendering that tree into the desired output format. The renderers provided by panprob are described below.

HTML

Renders a problem to HTML.

panprob.renderers.html.render(problem: Problem, overrides=None) → str

Render a problem to HTML.

This renderer is capable of rendering all node types from panprob.ast, except for the special nodes panprob.ast.Blob and panprob.ast.ParBreak, which should not appear in the AST.

Parameters:

  • problem (ast.Problem) – The problem to render.

  • overrides (dict, optional) – A dictionary mapping node types to functions that render them. If a node type is not in this dictionary, the default renderer is used. See below for the expected signature of these functions.

Returns:

The problem rendered as an HTML string.

Return type:

str

Raises:

RenderError – If there is an issue rendering this tree, such as when an unkown AST node type is encountered.

Notes

A renderer function is called with two arguments:

  1. The node to render.

  2. A function that recursively renders a node to HTML by choosing the correct node renderer for each child node.

It is expected to return an HTML string.

Gradescope Markdown

Renders a problem to Gradescope markdown.

panprob.renderers.gsmd.render(problem: Problem, overrides=None) → str

Render a problem in Gradescope markdown.

This renderer is capable of rendering all node types from panprob.ast except for panprob.ast.CodeFile and special nodes, like panprob.ast.Blob and panprob.ast.ParBreak.

panprob.ast.CodeFile is not supported because Gradescope markdown does not support code files. Gradescope markdown also does not support complex multiple choice options with multiple lines of text, so panprob.ast.Choice nodes with multiple lines of text will raise panprob.exceptions.RenderError.

Parameters:

  • problem (ast.Problem) – The problem to render.

  • overrides (dict, optional) – A dictionary mapping node types to functions that render them. If a node type is not in this dictionary, the default renderer is used. See below for the expected signature of these functions.

Returns:

The problem rendered as a markdown string.

Return type:

str

Raises:

RenderError – If there is an issue rendering this tree, such as when an unkown AST node type is encountered.

Notes

A renderer function is called with two arguments:

  1. The node to render.

  2. A function that recursively renders a node to markdown by choosing the correct node renderer for each child node.

It is expected to return an markdown string.
